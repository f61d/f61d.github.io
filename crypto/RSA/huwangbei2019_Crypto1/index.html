    huwangbei2019_Crypto¶

    Just decrypt it.

    Examination Site¶

    • RSA
    • multiplicative group modulo N

    Analysis¶

    The key code: 

    def gen():
    while True:
        g = getPrime(500)
        a = b = 0
        while not gmpy2.is_prime(2*g*a+1):
            a = random.randint(2**523, 2**524)
        while not gmpy2.is_prime(2*g*b+1):
            b = random.randint(2**523, 2**524)
        h = 2*g*a*b+a+b
        if gmpy2.is_prime(h):
            p = 2*g*a+1
            q = 2*g*b+1
            return p*q, g
    We will found that p,q have 1025 bit and have a common factor g which has 500 bit. Howevers,the a,b is unknown and we have to solve them.

    Reference: Further Attacks On Server-Aided Rsa Cryptosystems

    According to the article, we can fristly calculate the a*b and (a+b).

    As we have known N and g,we can calculate the u and v: 

    h = (N-1)/(g)
u = h/(g) # 4 * xy
v = h%(g) # 2 * (x+y)
    And now we have x + y = v + c*g and xy = u -c where c is the unknown carry in (N-1)/g in base g. Since The range of possible values for c is given cg < 4*(x+y) and x,y <N^(½)/g,so that there are of the order of C = sqrt(N)/(g*g) values of c to test.

    The article gives us a algorithm:Find a D being an integer larger than sqrt(C) and form the lists b0,bD,b2D,……,bD*D mod N and bu,bu-1,……,b^u-D we can sort these lists and find a common value b^rD ≡ b^u-D mod N and the c is rD+s!
